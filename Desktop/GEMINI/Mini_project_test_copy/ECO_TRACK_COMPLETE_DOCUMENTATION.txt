# EcoTrack Energy Management System - Complete Architecture Documentation

## 🏗️ System Architecture

EcoTrack is a **full-stack energy management system** that monitors and controls smart home devices to optimize energy consumption. The system follows a **client-server architecture** with three main components:

### 1. Backend (Flask API Server)
- **Technology**: Python Flask 3.0.3 + SQLite
- **Port**: 5000
- **Purpose**: RESTful API for data management and device control
- **Database**: SQLite (ecotrack.db) with 4 tables storing energy data, device profiles, events, and schedules

### 2. Frontend (Web Dashboard)  
- **Technology**: HTML5 + CSS3 + Vanilla JavaScript + Chart.js
- **Port**: 8000
- **Purpose**: Single-page application with real-time dashboard
- **Features**: 5 tabs (Dashboard, Cost, Appliances, Scheduler, Rooms)

### 3. IoT Integration (Tuya Smart Devices)
- **Platform**: Tuya Cloud API
- **Devices**: Smart plugs with energy monitoring
- **Protocol**: WiFi-based communication

## 🔄 Data Flow Architecture

Smart Plug (Physical Device) 
    ↓ (Power consumption data)
Tuya Cloud Platform
    ↓ (API calls with tinytuya)
Backend Flask API
    ↓ (RESTful responses)
Frontend Dashboard (Real-time UI updates)

## 📊 Core Features & Implementation

### 1. Real-Time Energy Monitoring
- **Data Collection**: Every 30 seconds, smart plugs send voltage and power readings
- **Mode Detection**: Devices classified as 'active', 'standby', or 'sleep' based on power thresholds
- **Storage**: SQLite database with timestamps and device metadata
- **Visualization**: Live charts showing consumption patterns

**Key API Endpoints**:
- GET /api/electricity/history?period=today|month|year
- POST /api/electricity/add (for manual data entry)

### 2. Smart Device Control
- **Instant Control**: Real-time on/off toggle via Tuya Cloud
- **Scheduling**: Time-based automation stored in localStorage
- **Device Detection**: Power signature analysis identifies appliance types
- **Status Monitoring**: Live device state tracking

**Control Flow**:
toggleDevice(deviceId, state) → 
    POST /api/devices/{id}/toggle → 
        Backend → Tuya Cloud → Smart Plug → 
            Response → UI Update

### 3. Cost Analysis & Savings
- **Energy Pricing**: $0.12/kWh calculation
- **Waste Detection**: Identifies sleep/standby energy waste
- **Savings Tracking**: Compares automated vs manual control
- **Projections**: Monthly cost predictions based on usage patterns

### 4. Multi-Tab Dashboard
- **Dashboard**: Overview with real-time metrics
- **Cost**: Detailed cost analysis and projections  
- **Appliances**: Individual device control and monitoring
- **Scheduler**: Time-based automation setup
- **Rooms**: Energy consumption by room/area

## 💾 Database Schema

The SQLite database contains 4 main tables:

1. **electricity_data**: Core energy readings (timestamp, device, voltage, power, mode)
2. **device_profiles**: Device configurations (type, rated power, sleep thresholds)  
3. **device_events**: Automation events and savings tracking
4. **schedules**: (Future use) Persistent schedule storage

### Table Structures:

CREATE TABLE electricity_data (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    device_name TEXT NOT NULL,
    mode TEXT NOT NULL,
    voltage REAL NOT NULL,
    power_watts REAL NOT NULL,
    timestamp TEXT NOT NULL,
    auto_controlled BOOLEAN DEFAULT FALSE,
    manual_override BOOLEAN DEFAULT FALSE,
    device_type TEXT DEFAULT 'unknown',
    rated_power REAL DEFAULT 0
);

CREATE TABLE device_profiles (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    device_name TEXT UNIQUE NOT NULL,
    device_type TEXT NOT NULL,
    rated_power REAL NOT NULL,
    sleep_threshold REAL NOT NULL,
    created_at TEXT NOT NULL
);

CREATE TABLE device_events (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    device_name TEXT NOT NULL,
    event_type TEXT NOT NULL,
    power_watts REAL NOT NULL,
    duration_minutes INTEGER NOT NULL,
    cost_wasted REAL NOT NULL,
    timestamp TEXT NOT NULL,
    auto_saved BOOLEAN DEFAULT FALSE
);

## 🔄 Real-Time Operation Cycle

1. **Startup**: Backend initializes database, frontend loads dashboard
2. **Data Refresh** (every 30 seconds):
   - Fetch latest energy readings from API
   - Update charts and metrics
   - Check and execute scheduled tasks
   - Update device status indicators
3. **User Interactions**:
   - Tab switching triggers specific data loads
   - Device control sends immediate API requests
   - Schedule creation stores in localStorage
4. **Background Processes**:
   - Tuya integration continuously polls smart plugs
   - Backend stores historical data
   - Frontend maintains real-time clock and date

### JavaScript Key Functions:
- setupTabSwitching(): Manages navigation between dashboard sections
- fetchElectricityData(): Retrieves energy consumption data
- updateApplianceDisplays(): Updates device status indicators
- refreshDashboard(): Main dashboard update routine
- toggleDevice(): Sends device control commands

## 🛡️ Security & Reliability

- **CORS Configuration**: Only allows frontend (localhost:8000)
- **Input Validation**: All API endpoints validate required fields
- **Error Handling**: Comprehensive try-catch blocks with user feedback
- **Data Integrity**: SQL parameterization prevents injection
- **Credential Security**: Tuya API keys stored in local JSON file

### Security Measures:
Authentication:

## 📋 API Endpoints Documentation

GET /api/devices

POST /api/devices/{id}/toggle

GET /api/electricity/history?period=today|month|year
  Response: { "status": "success", "data": [...] }
POST /api/electricity/add
